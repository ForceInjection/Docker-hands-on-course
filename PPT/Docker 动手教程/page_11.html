镜像构建与发布

镜像构建与发布流程

1. 构建镜像

docker build -t my-web-app:v1.0 .

2. 测试镜像

docker run -d -p 8080:80 --name my-app my-web-app:v1.0
# 测试访问
curl http://localhost:8080

3. 标记镜像

docker tag my-web-app:v1.0 your-username/my-web-app:v1.0
docker tag my-web-app:v1.0 your-username/my-web-app:latest

4. 推送镜像

docker login
docker push your-username/my-web-app:v1.0
docker push your-username/my-web-app:latest

多阶段构建优化

Dockerfile 多阶段构建示例

# 构建阶段
FROM node:16-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production

# 运行阶段
FROM nginx:alpine
COPY --from=builder /app/dist /usr/share/nginx/html
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]

优化技巧

  • 使用 alpine 基础镜像，减小体积
  • 多阶段构建，分离构建环境和运行环境
  • 清理缓存文件，减少最终镜像大小
  • 合理使用标签，便于版本管理和更新

发布注意事项

  • 确保镜像已通过本地测试
  • 使用语义化版本号（如 v1.0.0）
  • 为镜像添加适当的标签（latest、版本号）
  • 验证发布成功：访问 Docker Hub 仓库页面
